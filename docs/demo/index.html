Grid demo

Basic styles and structure

In its most basic form, chaestli needs a row and a column element to take action.
Containers are not necessary in any way.

?

Fixed container size

When you need a fixed layout, you can implement this by using a container with a constrain.
A container is also necessary when defining the space between the edges and the beginning of the grid.

?

Column size

Cols can be given a size by using the `grid--span` mixin with a given ratio that is a number between 0 and up to 1.

Basic spacing

The gutter between columns can be defined by using the `grid--space` and providing a `gutter` property.

Push and pull

The helpers `grid--push` and `grid--pull` can be used to give offset to specific columns

Four side spacing

The `gutter` property may contain up to four numbers that are interpreted identically to the `margin` or `padding` css property.

Edge spacing

The `edge` property in the config object given to the `grid--space` mixin defines the edge spacing.

Spacing (gutter and edge) change in response to media queries

The `grid--space` mixin takes an optional `mq` property that defines, when the gutter is applied.
This allows for complex changes in gutter and edge sizing according to the current media query.

Span changes in response to media queries

Also the `grid--span` mixin accepts an `mq` property that controls, when the sizing directive shall be active.

Nested grids

It's possible to nest a grid structure within a grid column. These grids can even have their own rules such as different gutters.

Important: Don't try to nest container elements, as their purpose (max-width, edge spacing) only makes sense in their outermost instance.
